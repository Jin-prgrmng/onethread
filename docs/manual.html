User Guide

OneThread 使い方ガイド

シングルタスク集中支援ツール — 今やることは、常に1つだけ。

1. アプリにアクセスする

OneThreadはインストール不要のWebアプリです。以下のURLをブラウザで開くだけで使えます。

アクセスURL

https://jin-prgrmng.github.io/onethread/

データはブラウザ内に保存されます。ログインや登録は不要です。

初回アクセスの手順

  1. 上記URLをブラウザのアドレスバーに入力してアクセスする
  2. フォーカスビュー中央の入力欄に「今やること」を入力する
  3. Enter キーまたは「開始」ボタンを押す
  4. タスクが大きく表示されたら準備完了です
ヒント： スマートフォンでも使えます。ホーム画面に追加しておくとアプリのように起動できます（iOSは「共有 → ホーム画面に追加」、Androidは「メニュー → ホーム画面に追加」）。

ソースコード・開発環境

GitHubリポジトリは以下で公開しています。ローカルで動かす場合は npm run dev を実行してください。

https://github.com/Jin-prgrmng/onethread

2. 基本的な考え方

OneThreadは「全部考えてよい、だが今やるのは1つだけ」という原則をUIで実現しています。

作業中に浮かんだことはすぐメモして「保留バッファ」に放り込み、今の作業には戻る。整理・分類・優先度付けは後でまとめてやる。このサイクルで、思考のオーバーフローを防ぎながら作業を前に進めます。

設計の意図：「整理してから始める」という思考パターンを断ち切るために、未整理のまま保留できる仕組みを中心に置いています。完璧なリストを作る前に、まず1つ動かすことが目的です。

4. 基本操作

タスクを始める

フォーカスビュー中央の入力欄に今やることを1行で入力して Enter または「開始」を押します。

タスクが画面に大きく表示されたら、それだけに集中してください。

タスクを終えたとき

3つのボタンで次の行動を選びます。

ボタン 意味 その後
完了 やり遂げた ログに記録され、キューの先頭が次のタスクとして表示される
スキップ 今じゃない このタスクはキューの末尾に回り、次のタスクに移る
保留 後で考える 保留バッファに移動し、次のタスクに移る

浮かんだことをメモする

作業中に別のことを思いついたときは、作業を止めずに即記録できます。

操作方法
キーボード m キーを押す（画面のどこにいても起動）
ボタン ヘッダー右の「メモ +」をタップ

入力して Enter を押すと即保存され、モーダルが閉じます。保存先は「保留バッファ」です。今は内容を考えなくて大丈夫です。

入力欄のラベル：「今は考えなくていい、ただ記録する」と表示されています。分類や整理は後でまとめてやれば十分です。

タイマーを使う

「とりあえず5分だけ手をつけてみる」という取り掛かりを支援します。

  1. フォーカスビューの「▶ タイマー開始」ボタンを押す
  2. カウントダウンが始まる（MM:SS 形式で表示）
  3. 終了すると「完了！」と画面上に表示される（音は鳴りません）
  4. 「リセット」ボタンで途中終了・再設定できる

タイマーの時間は ≡（設定）から 5 / 10 / 15 / 25 分に変更できます。デフォルトは5分です。

5. 下部の3つのセクション

ヘッダー部分をタップ/クリックすると開閉します。デフォルトでは折りたたまれており、件数だけ表示されます。

次のタスク（キュー）

現在のタスクが終わると、キューの先頭が自動的にフォーカスビューに繰り上がります。

操作方法
順番を変える 行をドラッグして並び替える
今すぐやる 「今すぐ」ボタンを押す（現在のタスクはキュー末尾へ）
タスクを追加 セクション下部の入力欄から追加

保留バッファ

「メモ +」で記録したことと、「保留」ボタンで保留にしたタスクが、すべてここに入ります。処理のタイミングは自由です。毎日の終わりや余裕があるときにまとめて見直すのが自然な使い方です。

ボタン動作
今すぐ 現在のタスクとして割り込む（緊急対応に）
キューへ 次のタスクリストの末尾に追加する
削除 不要だったものを消す（何もしないまま放置も可）

今日の完了

「完了」ボタンを押したタスクが完了時刻付きで並びます。今日分のみ表示されます。グラフや統計はありません。

6. 典型的な使い方の流れ

  • 1 朝、今日やることをキューに順番に並べる
  • 2 先頭のタスクが自動でフォーカスビューに表示される
  • 3 気が乗らなければ「▶ タイマー開始」でとりあえず5分だけ手をつける
  • 4 作業中に別のことを思いついたら m キーで即メモ → 忘れて作業に戻る
  • 5 タスクが終わったら「完了」→ 次が自動で表示される
  • 6 今日中にやらなくていいと気づいたら「スキップ」か「保留」
  • 7 1日の終わりに保留バッファを開いて、明日のキューに昇格させるか削除する
整理より前進を優先する： 完璧なリストが作れなくてもOKです。とりあえず1つ入力してタイマーを押すことが、このアプリの正しい使い方です。

7. データについて

タスクのデータはすべてお使いのブラウザ内（localStorage）に保存されます。サーバーへの送信や外部への同期は一切行いません。

項目内容
保存場所 ブラウザの localStorage（キー名: onethread_state）
ページリロード データは消えません
別のデバイス 同期されません（デバイスごとに独立したデータ）
ブラウザのデータ削除 「サイトのデータを削除」を実行するとデータが消えます
